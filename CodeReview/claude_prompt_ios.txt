### 特别关注的风险类型
==================================================

#### Swift / SwiftUI
- 强制解包（`!`）
- 强制类型转换（`as!`）
- 数组 / 字典越界访问
- UI 在非主线程更新
- SwiftUI 状态异常：
  - `@State / @Binding / @ObservedObject / @StateObject / @EnvironmentObject` 使用不当
  - 状态来源不清晰或多重状态源
  - 状态变化可能引起无限刷新
- 循环引用：
  - closure 捕获 `self`
  - `ObservableObject` 生命周期异常
- 值类型（struct）被误用为引用语义
- 错误处理缺失：
  - `try?` 吞掉关键错误
  - `throws` 未正确处理或上抛

#### Objective-C
- 内存泄漏（retain cycle）
- block / delegate 循环引用
- delegate 未声明为 `weak`
- KVO / Notification 未移除
- 异常处理缺失
- C 指针越界 / 野指针风险

#### 多线程 / 并发（GCD / Swift Concurrency）
- 非主线程更新 UI
- 主线程执行耗时操作（IO / 网络 / 大量计算）
- `dispatch_sync` 潜在死锁
- 多线程访问共享可变数据导致竞态
- Swift Concurrency：
  - `Task` 生命周期不可控
  - `@MainActor` 使用不合理
  - `async let` 未正确等待

#### 性能与资源
- 循环中执行 UI 或重计算逻辑
- 不必要的对象频繁创建
- 文件 / 网络操作缺乏异常处理
- 资源（timer / observer / animation）未释放

#### API 与系统兼容性
- 使用弃用 API
- 未做系统版本可用性判断（availability）
- 使用私有或不稳定行为

## 三、语言与框架专项审查要点（必须逐条对照）
==================================================

### 3.1 Swift 语言专项
- `guard let / if let` 是否存在分支遗漏
- `defer` 是否可能导致资源释放顺序错误
- `lazy` 属性是否在多线程环境被访问
- 错误传播路径是否完整清晰

### 3.2 SwiftUI 专项
- 是否在 `body` 中执行复杂计算
- `ForEach` 是否使用稳定、唯一的 `id`
- 是否在 `init / onAppear / task` 中执行不恰当的副作用
- View 与 ViewModel 是否职责清晰

### 3.3 Objective-C 专项
- ARC 环境下是否仍存在手动管理风险
- block 是否捕获 `self`
- delegate / notification 生命周期是否安全

### 3.4 并发模型
- 是否存在隐式跨线程调用
- 可变状态是否具备明确的线程保护策略

## 五、风险等级判定标准（用于统一严重程度）
==================================================
- 高  
  - 明确可能导致 crash、数据损坏、严重 UI 错乱
  - 明确的内存泄漏、死锁、线程安全问题
- 中  
  - 潜在性能问题
  - 并发使用不当但不一定立即触发
- 低  
  - 可维护性、可读性问题
  - 潜在但不紧急的隐患
